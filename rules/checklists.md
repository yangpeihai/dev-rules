# 检查清单

## 提交代码前

- [ ] 代码能够本地编译/构建通过
- [ ] 单元测试全部通过
- [ ] 静态代码检查无严重错误
- [ ] 代码格式化符合规范
- [ ] 提交信息符合规范（Conventional Commits）
- [ ] 关联的需求/缺陷单号已标注
- [ ] 敏感信息已移除
- [ ] 调试代码已清理
- [ ] 分支命名符合规范

---

## 发布前

- [ ] 所有代码审查通过（至少 2 个 Approve）
- [ ] 所有 CI/CD 检查通过
- [ ] 测试覆盖率满足要求（≥80%）
- [ ] 安全扫描无高危/严重漏洞
- [ ] 性能测试通过
- [ ] 文档已更新
- [ ] 回滚方案已准备
- [ ] 所有 [MUST] 级别评论已解决

---

## Code Review 六维度

1. **功能正确性** - 代码是否按预期工作
2. **代码风格** - 是否符合团队规范
3. **安全性** - 是否存在安全漏洞
4. **性能** - 是否有性能问题
5. **可维护性** - 代码是否易读、易修改
6. **测试覆盖** - 是否有充分的测试

---

## Review 评论分级

| 级别 | 说明 |
|------|------|
| [MUST] | 必须修复，否则不能合并 |
| [SHOULD] | 应该修复，除非有充分理由 |
| [NICE] | 建议改进，可选 |
| [Q] | 疑问，需要讨论 |
