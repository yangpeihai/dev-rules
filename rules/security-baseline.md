# 安全开发基线

> 配合 `secure-development` skill 使用

## 输入与输出

- ✅ 所有输入必须验证（白名单策略）
- ✅ 数据库查询必须参数化（禁止 SQL 拼接）
- ✅ 所有输出必须编码（防 XSS）

## 认证与授权

- ✅ 密码必须使用 bcrypt/argon2 哈希
- ✅ JWT Token 过期时间≤15 分钟
- ✅ 会话超时：30 分钟空闲 / 24 小时绝对
- ✅ Cookie 设置 HttpOnly、Secure、SameSite
- ✅ CSRF Token 验证（状态改变操作）
- ✅ 实施 RBAC 权限模型
- ✅ 资源级别鉴权
- ✅ 最小权限原则

## 密钥管理

- ✅ 禁止硬编码密钥（从环境变量读取）
- ✅ 禁止使用弱加密算法（MD5、SHA1）
- ✅ 禁止使用弱签名算法

## 日志安全

- ✅ 禁止记录敏感信息（密码、token、手机号、邮箱）
- ✅ 错误信息不暴露系统细节（堆栈、SQL）

## API 安全

- ✅ 实施速率限制
- ✅ 文件上传：类型和大小限制
- ✅ 外部调用设置超时

---

## 常见错误码

| 错误码 | 说明 |
|--------|------|
| 400001 | 通用参数错误 |
| 400002 | JSON 解析错误 |
| 401001 | Token 过期 |
| 401002 | Token 无效 |
| 403001 | 权限不足 |
| 404001 | 资源不存在 |
| 409001 | 资源冲突 |
| 429001 | 请求过多（限流） |
| 500001 | 内部服务器错误 |
