# 认证与安全规范

## JWT 规则

- 统一封装 token 生成、解析、刷新逻辑。
- Secret 或签名私钥不得写死在代码中。
- Claims 只存放必要身份信息，不塞入权限矩阵或敏感资料。
- 过期时间、刷新策略、签名算法必须配置化。

## 认证与授权边界

- Auth 中间件负责确认“你是谁”。
- Logic 或专门授权层负责确认“你能做什么”。
- 角色、租户、资源范围等授权规则不应耦合到通用 JWT 解析代码。

## 安全规则

- 密码必须使用安全哈希算法存储。
- 敏感字段在日志和响应中必须脱敏。
- 上传文件必须校验类型、大小、存储路径与访问权限。
- 对输入做长度限制、白名单校验和必要的转义，防止注入与滥用。

## 推荐实践

- 使用 request id 关联安全审计日志。
- 对登录、验证码、密码重置、Token 刷新接口增加限流和告警。
- 对后台管理接口补充二次校验或更严格的会话策略。

## 禁止事项

- 禁止：明文保存密码。
- 禁止：在日志中打印 Token、密码、身份证号等敏感信息。
- 禁止：仅依赖前端做权限控制。
- 禁止：把完整用户对象塞进 JWT Claims。
