# 配置与启动规范

## 启动顺序

1. 加载配置
2. 初始化日志
3. 初始化数据库与外部依赖
4. 注册 validator、自定义中间件、路由
5. 启动 HTTP Server

## Viper 规则

- 配置结构体集中定义，避免零散读取。
- 区分本地、测试、生产环境配置。
- 敏感配置优先从环境变量读取，再映射到结构体。
- 配置默认值必须显式声明，不依赖“未配置即零值”的偶然行为。

## Bootstrap 规则

- 依赖初始化集中在 `bootstrap/`，不要在业务代码中懒加载。
- 初始化失败必须立即中止启动，不带病运行。
- Server 的超时、端口、可信代理、优雅关闭策略必须配置化。

## 推荐实践

- 使用单独的 `HTTPConfig`、`DBConfig`、`JWTConfig` 等结构体分域管理配置。
- 启动日志打印关键非敏感配置，如端口、环境、版本、启用的中间件。
- 把 `gin.SetMode`、`SetTrustedProxies`、`http.Server` 超时等基础设置纳入启动规范。

## 禁止事项

- 禁止：在业务代码中直接读取配置文件。
- 禁止：组件按需临时初始化。
- 禁止：将密钥、数据库密码写死在仓库文件中。
