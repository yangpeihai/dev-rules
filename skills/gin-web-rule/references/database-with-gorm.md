# GORM 数据库规范

## Repository 规则

- Repository 方法第一参数统一使用 `context.Context`。
- 所有查询通过注入的 `*gorm.DB` 执行，不在方法内部重新创建连接。
- 事务由 Service 发起，Repository 接收事务句柄或可替换的 DB 抽象。

## 模型规则

- 显式声明主键、索引、唯一约束与表名策略。
- 审计字段统一命名，如 `created_at`、`updated_at`、`deleted_at`。
- 对 JSON、枚举、软删除字段建立清晰约定，不依赖隐式行为。

## 查询规则

- 列表查询必须有分页、过滤或上限。
- 排序字段必须做白名单控制。
- 预加载关联时明确范围，避免 N+1 和无界预加载。

## 推荐实践

- 将查询条件封装为 filter struct，避免 SQL 条件散落。
- 对热点写操作结合唯一约束和事务保证一致性。
- 对批量更新、批量删除显式加条件并记录影响行数。

## 强制规则

- Repository 只负责数据访问，不负责业务编排。
- 数据库错误在 repository 或 service 层转换成可识别错误类型。
- 所有 DB 操作必须可被请求上下文取消。

## 禁止事项

- 禁止：Repository 中编排业务逻辑。
- 禁止：无分页读取大结果集。
- 禁止：在事务内执行长耗时外部 HTTP/RPC 调用。
- 禁止：依赖默认排序返回“最新数据”。
