# 请求与响应规范

## 对象分层

- `Request DTO`：接收入参，承载绑定与校验 tag。
- `Entity/Model`：数据库实体，服务于持久化。
- `Response VO`：面向前端返回，屏蔽内部字段。

## 统一返回体

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "2f2f8404f8b9"
}
```

## 分页返回体

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "list": [],
    "total": 0,
    "page": 1,
    "pageSize": 20
  },
  "requestId": "2f2f8404f8b9"
}
```

## 强制规则

- JSON 字段统一使用小驼峰或团队既有风格，不混用下划线与驼峰。
- 成功响应与错误响应的顶层结构必须一致。
- 响应对象必须显式挑选字段，不直接暴露数据库字段、软删除标记、内部备注。
- 时间字段必须统一时区策略和序列化格式。
- 空列表返回 `[]`，不要返回 `null`。

## 推荐实践

- 对分页接口统一定义 `PageRequest` 与 `PageResponse`。
- 对枚举字段统一输出稳定值，不暴露内部魔法数字。
- 对兼容性要求高的接口，新增字段时保持向后兼容，避免重命名或改变语义。

## 禁止事项

- 禁止：直接返回 GORM Model 或嵌套懒加载对象。
- 禁止：同一服务内存在多套响应结构。
- 禁止：把底层错误详情直接透给客户端。
