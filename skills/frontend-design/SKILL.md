此技能指导创建独特、生产级的前端界面，避免通用"AI 敷衍作品"的美学风格。实现可工作的代码，对美学细节和创意选择保持卓越的关注。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能会包含有关目的、受众或技术约束的背景信息。

## 设计思维

编写代码前，先理解上下文并承诺一个大胆的审美方向：
- **目的**：这个界面解决什么问题？谁使用它？
- **基调**：选择一个极端方向：极简主义、极繁主义、复古未来主义、有机/自然风格、奢华/精致、俏皮/玩具感、编辑/杂志风格、粗野主义、装饰艺术/几何风格、柔和/粉彩、工业/实用主义等。有众多风格可供选择，从中汲取灵感，但设计出真正符合审美方向的作品。
- **约束**：技术需求（框架、性能、可访问性）。
- **差异化**：什么让这个界面令人难忘？有什么是用户会记住的独特之处？

**关键**：选择一个清晰的概念方向，并精准执行。大胆极繁主义和精致极简主义都可行——关键是意图性，而非强度。

然后实现可工作的代码（HTML/CSS/JS、React、Vue 等），它应该：
- 生产级且功能完善
- 视觉冲击力强且令人难忘
- 具有清晰审美立场的凝聚力
- 每个细节都精心打磨

## 前端美学指南

关注以下方面：
- **排版**：选择美观、独特且有趣的字体。避免通用字体如 Arial 和 Inter；转而选择能提升前端美学的独特字体；富有特色的字体选择。将独特的展示字体与精致的正文字体搭配使用。
- **色彩与主题**：坚持统一的审美风格。使用 CSS 变量保持一致性。主色调搭配鲜明的强调色，胜过保守、均匀分布的调色板。
- **动效**：使用动画创造效果和微交互。对于 HTML 优先使用纯 CSS 方案。如使用 React 则选用 Motion 库。聚焦高影响力时刻：一个精心编排的页面加载配合错落的渐显动画（animation-delay），比零散的微交互更能带来愉悦感。使用滚动触发和悬停状态带来惊喜。
- **空间构成**：意想不到的布局。不对称。重叠。对角线流。打破网格的元素。宽裕的负空间或受控的密度。
- **背景与视觉细节**：营造氛围和深度，而非默认使用纯色。添加与整体审美匹配的上下文效果和纹理。应用创意形态，如渐变网格、噪点纹理、几何图案、分层透明度、戏剧性阴影、装饰性边框、自定义光标和颗粒叠加。

切勿使用通用 AI 生成的美学风格，如过度使用的字体家族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏情境特色的公式化设计。

以创意方式解读，做出令人惊喜的选择，让设计感觉真正为情境而生。每个设计都应不同——变换明暗主题、不同字体、不同美学风格。永远不要在不同代之间收敛于常见选择（比如 Space Grotesk）。

**重要**：实现复杂度需匹配审美愿景。极繁主义设计需要精细的代码，包含大量动画和效果。极简主义或精致设计则需要克制、精准，以及对间距、排版和微妙细节的细致关注。优雅源于愿景的完美执行。
